端智能日报 2026-04-30

本期聚焦：工业级生成式推荐"三连发"——美团、京东同期实现 LLM 驱动生成式排序/重排的大规模线上验证；与此同时，LLM 中间层表示压缩为广告 CTR 模型提供了成本最低的语义增强路径；端侧硬件侧则迎来首个 SSM + RRAM 计算存储一体化的 Nature 级别验证。

1. 【MRC (arXiv 2604.18146)：发现 LLM 中间层表示优势（MRA），HSIC 压缩框架强化工业 CTR 模型】

  • 评分：A/5 · I/4 · S=14 · P0 · 价值类型：收入提升
  • 来源：多机构 | arxiv.org/abs/2604.18146 | 2026-04-20（v2 2026-04-21）

内容总结（~130 字）： 论文揭示"中间层表示优势"（Mid-layer Representation Advantage, MRA）：LLM 中间层特征在下游推荐任务中系统性优于最终层，作者基于模块化理论解释为最终层被迫专注代理任务导致性能退化。提出 Modular Task Decoupling 方法，以 HSIC 约束和信息瓶颈框架将 LLM 中间层表示高效压缩，注入下游 CTR 模型（DCNv2）。规模实验显示更大的 LLM 带来更大收益，并通过线上 A/B 实测验证业务指标改善。工程要点：LLM 仅需离线推理一次，提取中间层特征后存入特征系统，不改变在线服务架构，计算成本可接受。

业务价值与适配性： 可以让我们在用户/素材 Embedding 接入 CTR 模型时，将 LLM 特征层的选取从默认"最终层"改为"中间层"，成本几乎为零，可能带来显著精度提升。可直接应用于我们现有 LLM-for-CTR Pipeline 的特征层选取优化，建议一周内跑一次小规模 A/B 验证。

2. 【GenRec (arXiv 2604.14878)：京东 App 偏好导向生成式推荐——Page-wise NTP + Token Merger 2× 压缩 + GRPO-SR 强化对齐，月级 A/B 实测 +9.5% 点击 / +8.7% 成交】

  • 评分：A/4 · I/4 · S=12 · P0 · 价值类型：收入提升
  • 来源：京东 / arXiv | arxiv.org/abs/2604.14878 | 2026-04-14

内容总结（~140 字）： 针对 decoder-only 生成式推荐三大工程痛点，提出 GenRec：① Page-wise NTP 按页整体监督而非逐 item，密化梯度信号，消除一对多歧义；② 非对称线性 Token Merger 在 Prompt 侧将多 token Semantic ID 压缩约 2× 输入长度，解码侧保持全分辨率，精度损失可忽略；③ GRPO-SR 以带 NLL 正则的组相对策略优化做强化对齐，将用户真实行为信号对齐进生成策略。京东 App 真实流量月级 A/B 测试：点击量 +9.5%，成交量 +8.7%。参数量级未披露，但 Token Merger 压缩使推理成本贴近非生成式基线。

业务价值与适配性： 可以让我们在广告精排/重排阶段以可控推理成本部署生成式位序优化模型，并借助 GRPO-SR 将成交/点击等延迟反馈对齐进模型策略。Page-wise NTP 的列表级训练目标设计对广告多位组合优化（如横幅 + 插屏混排）有直接借鉴价值，可在现有重排器升级时纳入方案比较。

3. 【NSGR (arXiv 2604.05314)：美团 Next-Scale 树形生成式重排——Coarse-to-Fine 多尺度生成，解决全局/局部失衡与训练-评估目标不一致，已上线】

  • 评分：A/4 · I/4 · S=12 · P0 · 价值类型：收入提升/用户体验
  • 来源：美团 / arXiv | arxiv.org/abs/2604.05314 | 2026-04-07

内容总结（~130 字）： 提出"逐尺度生成"新范式 NSGR：不同于自回归（逐 item）或非自回归（一步全出），NSGR 以树状结构从用户兴趣粗粒度到细粒度渐进展开推荐列表。核心组件 Next-Scale Generator（NSG）在每一层级做 beam 扩展，平衡全局多样性与局部相关性。同步引入 evaluator 协同，修复现有生成重排器的 goal inconsistency 问题（generator 与 evaluator 训练目标不一致）。已在美团食品外卖平台线上部署，服务亿级用户。推理侧复杂度通过分层 beam-search 可控，硬件需求与当前列表模型相近。

业务价值与适配性： 可以让我们在广告 Rerank 阶段同时优化列表级指标（如页面整体 GMV）与 item 级 CTR，不依赖超大云模型。NSGR 对"自回归重排器训练-推理 gap"问题的解法可直接借鉴至我们现有广告重排实验：若已在用自回归重排器，goal inconsistency 修复是低成本高价值的改进点。

4. 【MBGR (arXiv 2604.02684)：美团多业务生成式推荐——BID 跨域 Tokenization + MBP 多任务预测头 + LDR 标签动态路由，生产上线】

  • 评分：A/4 · I/4 · S=12 · P0 · 价值类型：收入提升
  • 来源：美团 / arXiv | arxiv.org/abs/2604.02684 | 2026-04-03

内容总结（~130 字）： 针对多业务场景（外卖/买菜/闪购）共享 Semantic ID 导致的表示混淆（seesaw phenomenon）和 NTP 无法建模跨业务行为的痛点，提出 MBGR 三件套：① BID（Business-aware Semantic ID） 感知业务域的 tokenization，保持语义完整性并区分跨域语义；② MBP（Multi-Business Prediction） 多任务预测头，提供业务专属解码能力；③ LDR（Label Dynamic Routing） 将稀疏多业务标签路由为稠密标签，改善训练信号质量。离线和线上实验均在美团外卖平台生产验证，已上线服务。无公开延迟数据，但框架设计保持 NTP 推理范式兼容。

业务价值与适配性： 可以让我们在音频广告"多场景统一排序"（如 podcast 广告 vs 短视频广告 vs 开屏广告）需求下，用 MBGR 的 BID+LDR 方案统一 tokenization 同时维持场景差异性。LDR 的稠密标签路由思路对"多业务稀疏标签导致训练不稳定"这一常见工程痛点有直接参考价值。

5. 【Network Edge Inference for LLMs (arXiv 2604.22906)：端侧 LLM 推理综述——系统架构、模型优化、资源调度全景图谱】

  • 评分：A/3 · I/3 · S=9 · P1 · 价值类型：新能力探索
  • 来源：多机构（英、美、新加坡） / arXiv | arxiv.org/abs/2604.22906 | 2026-04-24

内容总结（~120 字）： 系统综述在网络边缘设备执行 LLM 推理的挑战与方法，覆盖三大模块：① 系统架构（单设备、协作推理、cloud-edge 分层 pipeline）；② 模型优化（量化 INT4/INT8/FP8、蒸馏、剪枝、投机解码、KV cache 压缩）；③ 资源管理与调度（延迟感知调度、能耗约束）。以 70B LLaMA fp16 需 140 GB 为基准量化挑战，对比 INT4（~35 GB）+ 投机解码等主流工程化方案。提交于 2026-04-24，是目前最新的端侧 LLM 工程化综述，作者团队覆盖英、美、新加坡学术机构。

业务价值与适配性： 可以让我们在广告端云协同架构选型时，使用该综述作为技术路线对比的"地图文件"——特别是在 INT4 vs INT8 量化精度损失、投机解码收益边界等决策节点上有参考基准。当前广告端侧 LLM 推理处于立项调研阶段，暂不适用生产，建议作为立项参考材料。

6. 【Michigan/Nature Communications：SSM + RRAM 计算存储一体化——首次将状态空间模型映射至 Resistive RAM 跨阵列，突破端侧实时推理功耗边界】

  • 评分：A/2 · I/4 · S=8 · P1 · 价值类型：新能力探索
  • 来源：University of Michigan / Nature Communications | nature.com/articles/s41467-025-68227-w | Nature Commun. vol.17, art.1513（2026-04）

内容总结（~120 字）： 首次将状态空间模型（SSM，Transformer 高效替代架构）映射至 RRAM 跨阵列计算存储一体化硬件，利用 RRAM 天然短时记忆衰减模拟 SSM 状态演化，实现"在存储单元中原地计算"。基于标准 65nm CMOS 工艺制造，向量矩阵乘误差仅 4.6 bits（接近理论值）；钨氧化物记忆单元衰减与模型预测高吻合。实时处理连续事件序列（视频流、传感器数据），延迟与功耗均显著优于传统数字硬件。目标部署：手机、助听器、自动驾驶摄像头。报道来源：TechXplore 2026-04。

业务价值与适配性： 与当前广告系统直接关联弱，暂不适用于近期开发。中长期价值：若 SSM + 神经形态硬件路线成熟，端侧用户行为序列建模（长序列 attention 是 CTR 模型的重要瓶颈）的功耗/延迟边界将大幅下降。建议列入 2-3 年技术雷达观察项。

其他观察

今日最显著趋势是生成式推荐的工业化浪潮：美团和京东在一个月内先后公开 MBGR（多业务统一 Tokenization）、NSGR（树形重排）、GenRec（强化对齐生成）三套系统，意味着生成式精排/重排已进入国内头部平台的稳定生产迭代期。与此同时，MRC 提供的"LLM 离线中间层表示 + CTR 注入"路线成本最低，适合作为短期切入点验证 LLM 对广告 CTR 的增益。
