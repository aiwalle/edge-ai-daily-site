端智能日报 2026-04-25

本期聚焦：ICLR 2026（里约，第三天）持续输出端侧压缩技术——ParoQuant 以 INT4 轮转量化将推理 LLM 精度超越 AWQ 2.4%；ONNX Runtime 1.25.0 同日（4/22）发布，为 Qualcomm QNN SDK 2.37 和 ARM64 INT4 提供生产级支持；Apple 在会场展示 M5 芯片 + MLX 端侧 LLM 完整离线推理能力。框架、量化算法、芯片三条线同步演进，端侧广告推理的部署基础设施正在快速闭环。

1. 【ONNX Runtime 1.25.0 发布：QNN SDK 2.37 集成、ARM64 非对称 INT4 MatMulNBits、ArmNN EP 正式退出】

  • 评分：A 3/5 · I 4/5 · S=10 · P1 · 价值类型：成本优化 / 新能力探索
  • 来源：Microsoft / GitHub | github.com/microsoft/onnxruntime/releases/tag/v1.25.0 | 2026-04-22

内容总结（140 字）： ONNX Runtime 1.25.0 于 2026-04-22 正式发布，移动端关键工程变化：① QNN SDK 2.37 集成——针对支持 SME2（Scalable Matrix Extension v2）的 Qualcomm 硬件增强 SGEMM、IGEMM、Dynamic Quantized MatMul 算子性能，Conv2D 在 Hexagon NPU 上效率显著提升；② ARM64 非对称 4-bit MatMulNBits——DQ→MatMulNBits 算子融合扩展至量化 LLM 的 CPU 推理路径，覆盖更广泛 ARM 设备（含 KleidiAI BF16 SME2 内核）；③ CUDA Plugin EP——首次允许第三方 GPU 后端作为动态加载插件接入，无需重新编译 ORT；④ ArmNN EP 正式移除，用户须迁移至 MLAS/KleidiAI CPU EP 或 QNN EP。C++20 编译环境、CUDA 12.0+ 为新强制要求。

业务价值与适配性： 可以让我们在 Qualcomm 骁龙旗舰设备（Hexagon NPU）上以 ONNX Runtime 原生路径部署广告 CTR/CVR 模型，QNN SDK 2.37 的 Conv2D NPU 加速可直接降低图文素材多模态编码延迟。可直接应用：升级至 ORT 1.25.0 + QNN SDK 2.37，将当前 CPU 推理路径迁移至 Hexagon NPU；ArmNN EP 存量用户需立即规划迁移至 QNN EP，否则无法跟进后续 ORT 版本的性能迭代。

2. 【ParoQuant（ICLR 2026）：INT4 成对轮转量化超越 AWQ 2.4%，原生支持 Apple Silicon MLX 和 NVIDIA vLLM】

  • 评分：A 3/5 · I 3/5 · S=9 · P1 · 价值类型：成本优化
  • 来源：Z Lab / ICLR 2026 | arxiv.org/abs/2511.10645 | 2026-04-25（ICLR 2026 Poster 展示）

内容总结（140 字）： ParoQuant（Pairwise Rotation Quantization）是 ICLR 2026 收录的 INT4 权重后训练量化（weight-only PTQ）方法。核心技术：Scaled Pairwise Rotation——通过可学习的独立 Givens 轮转变换配合逐通道缩放，将权重矩阵的通道离群值均匀分散，使标准 INT4 量化不再受离群值"拉高 scale"影响，同时保持硬件友好性，推理速度接近 AWQ。主要优势：在推理型 LLM（Qwen3-4B、DeepSeek-R1 等）上，AWQ 4-bit 因推理链路误差累积在 MMLU-Pro 上损失约 3%，ParoQuant 以相近速度消弭此差距，平均精度超越 AWQ 2.4%。开源支持：NVIDIA GPU（vLLM / Transformers）和 Apple Silicon（MLX），已提交 MLX-LM 集成 PR（#977），代码开源于 github.com/z-lab/paroquant。

业务价值与适配性： 可以让我们在 iOS 平台（iPhone A 系芯片 + MLX）以更高精度部署 4-bit 量化广告意图识别或用户兴趣 LLM 模型，相比 AWQ 在推理任务上提升 2.4% 精度，对广告相关性判断直接有收益。可直接应用：在端侧 LLM 量化管线中将 AWQ 替换为 ParoQuant（尤其涉及推理步骤的意图理解任务），Apple Silicon MLX 和 Android vLLM 两条路径均有官方支持，无需额外适配工作。

3. 【Apple at ICLR 2026：M5 Max MacBook + MLX 完整本地 LLM 推理 Demo，ParaRNN 获 Oral（训练加速 665×）】

  • 评分：A 2/5 · I 4/5 · S=8 · P2 · 价值类型：新能力探索
  • 来源：Apple Machine Learning Research | machinelearning.apple.com/updates/apple-at-iclr-2026 | 2026-04-23（ICLR 2026 展示中）

内容总结（130 字）： Apple 在 ICLR 2026 现场展示两项端侧推理能力：① MacBook Pro（M5 Max）+ MLX：在 Xcode 原生开发环境中运行量化前沿编程模型（frontier coding model）的完整本地离线推理，展示 M5 Max 统一内存架构的 LLM 推理上限；② SHARP + iPad Pro（M5）：图像输入至 3D Gaussian 点云生成的全流程端侧实时运行，体现多模态推理能力。学术成果：ParaRNN（Oral）提出非线性 RNN 并行训练方案，比传统顺序方法加速 665×，首次实现 RNN 的大规模语言模型训练，可作为比 Transformer 更节省 KV cache 的端侧序列建模架构候选。MLX 官方已支持 M5 Neural Accelerator GPU。

业务价值与适配性： 可以让我们预判 Apple Intelligence / iOS 27 将进一步开放本地 LLM 调用能力——若量化 coding-class 模型已在 M5 Max 本地跑通，iPhone 的广告意图推理端侧化路线将加速。暂不适用：当前展示为 MacBook 形态，手机端落地时间线待 WWDC 2026；但 ParaRNN 的轻量序列架构值得跟踪，作为未来 CTR 序列模型端侧候选（相比 Transformer 在自回归推理时无 KV cache 开销优势）。

4. 【Qualcomm AI Hub 近期更新：Galaxy S25 上线、DLC 原生编译支持、QNN FP32/FP16 输入自动对齐】

  • 评分：A 2/5 · I 3/5 · S=7 · P2 · 价值类型：成本优化 / 新能力探索
  • 来源：Qualcomm AI Hub | workbench.aihub.qualcomm.com/docs/hub/release_notes.html | 2026-04（持续更新）

内容总结（110 字）： Qualcomm AI Hub 发布多项工程更新：① Samsung Galaxy S25 系列正式加入可部署设备列表，开发者可直接在 Hub 完成 compile / profile / inference 三步工作流，无需本地设备；② DLC（Deep Learning Container）原生支持——可提交 .dlc 格式模型用于 Qualcomm NPU 编译与性能基准，省去 ONNX 转换中间环节；③ QNN FP16 I/O 兼容 FP32 输入——量化 QNN 模型推理时传入 FP32 数据自动无缝转换，降低集成复杂度。配合平台已有 Llama 3.2、Gemma 系列优化模型，整体部署流程更短。

业务价值与适配性： 可以让我们以 AI Hub 作为广告端侧模型的"性能 CI 平台"——每次模型更新，自动在 Galaxy S25（骁龙 8 Elite）等多款旗舰机型上运行 latency + accuracy 基准测试，代替手动设置开发环境，缩短端侧模型验证周期至分钟级。可借鉴思路：将 AI Hub 纳入广告端侧模型 CI/CD 流水线，DLC 路径与 ORT 1.25.0 QNN 路径形成互补——前者面向 Qualcomm 深度优化，后者面向跨平台兼容。
