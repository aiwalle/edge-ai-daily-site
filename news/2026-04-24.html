端智能日报 2026-04-24

本期聚焦：ICLR 2026（里约，第二天）密集发表端侧 AI 论文——SIREN 将 LLM 推荐推理延迟压缩 100 倍、NANOMIND 首次实现多模态模型跨 NPU/GPU/DSP 异构卸载；腾讯混元 3.0 同日发布并开源，MoE 推理效率提升 40%，云端大模型的稀疏化路线向端侧传导的势头愈发清晰。

1. 【SIREN（ICLR 2026）：LLM 推荐兴趣精炼 + 内化推理，Token 减 48.7%，推理延迟降 >100×】

  • 评分：A 4/5 · I 3/5 · S=11 · P1 · 价值类型：收入提升 / 成本优化
  • 来源：OpenReview / ICLR 2026 | openreview.net/forum?id=NVrXCKaEjM | 2026-04-23（ICLR 2026 发表）

内容总结（140 字）： SIREN 解决 LLM 推荐的两个核心工程痛点：① 用户历史上百条 item 导致 Input Token 过多；② CoT 推理需自回归生成 rationale，实际线上延迟不可接受。核心方法：长期兴趣精炼（Interest Sketch） 构建 Token 有界的紧凑用户偏好表示，保留持久信号并压制噪声；内化推理（Internalized Reasoning） 通过两阶段训练（rule-based RL 显式推理 → 隐式对齐）将 CoT 内化到模型参数，推理时无需生成 rationale token。实测：平均 Input Token 减少 48.7%，MAE 比 runner-up 下降 20.1%，推理延迟比 CoT-based LLM 推荐降低 >100 倍。无平台/硬件依赖，可对接任意 LLM 后端。

业务价值与适配性： 可以让我们在端侧或云端 LLM 精排场景中，把 LLM 推荐的延迟从秒级压缩到百毫秒级，同时以更少 Token 编码更准确的长期用户兴趣，直接提升广告相关性与 CTR。可直接应用：Interest Sketch 机制可迁移到广告用户兴趣建模（替代当前 flat 历史拼接），内化推理训练方法适用于端侧广告小模型微调；与 04-23 报道的 OD-LLM 形成互补——SIREN 解决输入 Token 效率，OD-LLM 解决模型体积压缩。

2. 【NANOMIND（ICLR 2026）：端侧多模态异构卸载框架，模块级 NPU/GPU/DSP 分配，电池设备可完整运行 LMM】

  • 评分：A 3/5 · I 4/5 · S=10 · P1 · 价值类型：新能力探索 / 成本优化
  • 来源：arXiv / ICLR 2026 | arXiv 2510.05109 | 2026-04-23（ICLR 2026 发表）

内容总结（140 字）： NANOMIND（论文题 "Tiny but Mighty"）是专为异构 SoC（NPU + GPU + DSP + unified memory）设计的多模态大模型（LMM）端侧推理框架。核心创新：将 LMM 拆分为功能模块（视觉编码器、投影层、语言模型主干、音频模块等），实施模块级动态卸载（module-level dynamic offloading）——每个模块调度到最适合的加速器，避免整体串行执行对统一内存带宽的浪费；结合定制化低比特 kernel 和系统级调度，在紧凑的电池供电设备上实现完整 LMM 全程离线运行，latency 和能耗均显著优于常规数字硬件。支持平台：现代 Android SoC 异构处理器。

业务价值与适配性： 可以让我们在中低端 Android 机型（NPU+GPU+DSP 异构）上以更低功耗运行多模态素材理解模型（图文/视频创意打分），覆盖超过 70% 广告受众的设备范围，同时避免热降频导致的推理不稳定。可借鉴思路：NANOMIND 的"模块粒度调度"适用于端侧广告推理流水线解耦——用户兴趣编码（NPU 友好）、创意多模态理解（GPU 友好）、CTR 打分（NPU/CPU）分别调度；与 04-23 报道的 MediaTek Dimensity 9500 + LiteRT 方案组合，可形成更完整的端侧多模态广告推理部署路径。

3. 【腾讯混元 3.0 正式发布并开源：295B 参数 / 21B 激活 MoE，推理效率整体提升 40%】

  • 评分：A 2/5 · I 4/5 · S=8 · P2 · 价值类型：成本优化 / 新能力探索
  • 来源：财联社 | 财联社报道 | 2026-04-23

内容总结（130 字）： 腾讯于 2026-04-23 正式发布并开源混元 3.0（Hy3 preview），为下一代 LLM 旗舰。规格：295B 总参数，21B 激活参数（MoE 稀疏架构，激活率约 7.1%），支持 256K 上下文长度。工程亮点：通过模型-推理框架深度协同实现整体推理效率提升 40%，成本较上一代显著降低；姚顺雨团队首次独立主导研发。发布形式为预览版，完整版时间表未公开。同期 benchmark 表现：中文推理、代码、数学均有提升，完整评测数据待公布。目前云端部署，端侧版本未宣布，MoE 稀疏激活路径为后续端侧化提供基础。

业务价值与适配性： 可以让我们在云端广告大模型（意图推理、创意生成蒸馏源）中以 40% 更低成本调用同等规模模型；MoE 稀疏化（295B 总参 / 21B 激活）的架构为后续向端侧 MoE 迁移提供开源基础。可借鉴思路：持续追踪混元 3.0 是否发布蒸馏版（compact dense），若有则可直接用于广告精排云端降本；"模型-框架协同"带来 40% 效率提升的工程细节值得跟进，方法论对端侧推理框架优化亦有参考价值。

4. 【MobileIPL（ICLR 2026，小米 AI Lab + NTU）：移动 Agent 迭代偏好学习，三大 GUI benchmark SOTA，超越 OS-ATLAS 和 UI-TARS】

  • 评分：A 2/5 · I 3/5 · S=7 · P2 · 价值类型：新能力探索
  • 来源：OpenReview / ICLR 2026（小米 AI Lab、南洋理工、人民大学）| OpenReview PDF | 2026-04-23（ICLR 2026 发表）

内容总结（120 字）： MobileIPL 由小米 AI Lab 联合 NTU、中国人民大学提出，针对移动端 GUI Agent 的思维链推理能力提升问题。方法：迭代偏好学习（Iterative Preference Learning）框架——通过 Monte Carlo Tree Search（MCTS）构建 CoaT-tree（思考+行动组合树），使用基于规则的奖励对每步推理打分（无需人工标注），生成 thinking-level DPO 训练对；配合三阶段指令演进防止 SFT 过拟合，让端侧 Agent 推理能力持续迭代提升。在 AndroidWorld、MobileBench、GUI-Odyssey 三个标准 benchmark 上超越 OS-ATLAS 和 UI-TARS，达到 SOTA。运行环境：端侧移动设备。

业务价值与适配性： 可以让我们在广告 A/B 测试自动化、端侧 App 内策略 Agent 等场景中，用更少人工标注数据训练出更强的移动端 UI 自主操控能力。暂不适用：MobileIPL 当前聚焦 GUI 操控任务（点击/滑动/表单填写），与广告排序核心链路距离较远；但其 MCTS + 规则奖励 + DPO 的端侧 Agent 数据自举范式，可借鉴到广告端侧策略 Agent 训练（少量专家标注 + 多轮 MCTS 探索生成训练对）。

5. 【Expedera Origin Evolution NPU IP 获 2026 Edge AI 年度最佳处理器 IP 奖：外存传输减 75%，单核扩展至 64 TFLOPS】

  • 评分：A 2/5 · I 3/5 · S=7 · P2 · 价值类型：成本优化
  • 来源：Expedera / Edge AI and Vision Alliance | Expedera 博客 | 2026-04-21

内容总结（120 字）： Edge AI and Vision Alliance 于 2026-04-21 公布 2026 年度 Edge AI 产品奖，Expedera 的 Origin Evolution NPU IP 获"最佳 Edge AI 处理器 IP"。核心架构：包式路由（packet-based routing）——将神经网络切成 packet 并路由到专用计算块，相比主流 NPU 减少主内存访问超过 75%，系统功耗降低、吞吐提升；单核算力可扩展至 64 TFLOPS（FP16/INT8）；面向手机场景：已量产部署于智能手机 SoC，支持 GenAI 模型（LLM/VLM）的边缘推理，memory 效率是其主要差异化卖点。IP 授权模式（非自研芯片，授权给 SoC 厂商集成）。

业务价值与适配性： 可以让我们在评估合作 OEM 设备端侧推理能力时，了解"包式路由 NPU"正在成为 SoC 厂商的差异化选项——内存带宽是端侧推理核心瓶颈，75% 外存传输减少若在广告模型上成立，将直接改善延迟和功耗。暂不适用：Expedera 是 IP 供应商，不直接影响框架/平台选型；但若合作 OEM 的 SoC 集成 Expedera IP，可在模型部署优化时将其 packet-routing NPU 特性纳入 kernel 调优考量。
