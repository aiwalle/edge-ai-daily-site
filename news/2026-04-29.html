端智能日报 2026-04-29

本期聚焦：LLM-HYPER 以 LLM 作为 hypernetwork 直接生成广告 CTR 冷启动参数（A5），是近期广告算法领域最值得快速评估的新范式；SAGER 提出每用户独立演化策略技能，补足当前推荐 Agent 最后一块短板；MNN 3.5.0 重磅更新以 TurboQuant + Vulkan LLM 夯实中文移动端推理基础设施，Samsung 旗舰机 multi-LoRA 一体化模型量产则将"端侧多任务切换"从理论推向商业落地。

1. 【LLM-HYPER：LLM 作为 Hypernetwork 直接生成 CTR 预估参数，无训练解决广告冷启动问题】

  • 评分：A5 · I3 · S=13 · P0 · 价值类型：收入提升
  • 来源：Walmart Global Tech | arxiv.org/abs/2604.12096 | 2026-04-13

内容总结（130 字）： LLM-HYPER 针对广告冷启动场景（新广告缺乏用户反馈）提出 training-free 解法：LLM 不做推理输出，而是作为 hypernetwork 直接生成线性 CTR 预估器的逐特征权重。流程：① 用 CLIP embedding 从历史广告库中检索语义相似的历史投放案例；② 将案例与目标广告的多模态内容（文本 + 图像）格式化为 few-shot CoT prompt 输入 LLM；③ LLM 推理用户意图、特征影响力和内容相关性，输出 CTR 权重向量；④ 加入归一化与校准确保生产级分布一致性。零训练开销，全流程推理时完成个性化权重生成。

业务价值与适配性： 可以让我们在新广告素材上线前 24 小时内无需等待 warm-up 数据即获得初始 CTR 估计，冷启动 RPM 损失有望大幅压缩。可直接应用：CLIP 语义检索历史投放案例 + LLM CoT 权重生成可直接作为现有 CTR 冷启动模块的即插即用替代方案；端侧实现路径：CLIP embedding 索引本地化 + SLM 替代 LLM 做权重生成，可实现隐私保护的端侧冷启动。

2. 【SAGER：每用户独立"策略技能"文档 + 自演化推荐 Agent——解决 LLM 推荐 Agent 的决策逻辑个性化短板】

  • 评分：A4 · I3 · S=11 · P1 · 价值类型：新能力探索
  • 来源：多机构 | arxiv.org/abs/2604.14972 | 2026-04-16

内容总结（130 字）： SAGER 指出当前 LLM 推荐 Agent 的核心矛盾：记忆（用户偏好）已个性化，但推理逻辑（system prompt）对所有用户共享且固化——当推荐失败时，Agent 只更新"知道了什么"，而不追问"为什么这样决策"。SAGER 为每个用户维护一份结构化自然语言 Policy Skill 文档，编码个性化决策原则（如"该用户拒绝推荐时通常是因为类型重复而非价格"）。失败时，Agent 以反事实归因自动更新该文档；文档通过蒸馏可被轻量化后下推至端侧。架构：Policy Skill + Memory 双轨并行，联合驱动推荐决策。多数据集上优于共享策略基线。

业务价值与适配性： 可以让我们在端侧广告推荐 Agent 中为每个用户维护一份轻量"广告策略偏好文档"（如"该用户对重复品类广告有强负反馈"），驱动端侧排序/过滤决策，无需上传原始行为即可实现个性化策略优化。可借鉴思路：Policy Skill 的"失败 → 归因 → 策略文档更新"闭环，可迁移为广告 SDK 的端侧负反馈学习机制；与 CDA4Rec（04-27 报道）的端侧摘要方案互补——SAGER 管"怎么推"，CDA4Rec 管"推什么"。

3. 【MNN 3.5.0（阿里巴巴）：Vulkan LLM 推理 + TurboQuant TQ3/TQ4 KV Cache 量化 + QNN/RISC-V 后端——移动端推理框架里程碑更新】

  • 评分：A3 · I4 · S=10 · P1 · 价值类型：成本优化
  • 来源：阿里巴巴 / GitHub | github.com/alibaba/MNN/releases | 2026-04 发布

内容总结（140 字）： 阿里巴巴 MNN 3.5.0 围绕三条主线重磅更新：① 多后端 LLM 推理扩展：Vulkan 后端新增完整 LLM 推理支持（扩大安卓 GPU 覆盖）；摩尔线程 MUSA 后端接入；QNN 后端新增 Attention 算子与更多 LLM 算子支持；RISC-V RVV 向量扩展落地（覆盖 RISC-V 边缘设备）。② 高性能量化与采样优化：全新 TurboQuant TQ3/TQ4 KV Cache 量化方案（对标 Google TurboQuant 路线，可将 KV Cache 显存压缩至约 4-bit 级别）；Sampler 重构为 Pipeline 架构；多轮对话 Prompt Cache 支持。③ 端侧语音交互增强：异步 3 阶段 Token2Wav 流水线（DiT ∥ Vocoder）；AEC 智能打断。MNN 作为国内最成熟的移动端推理引擎（已覆盖淘宝、支付宝等亿级 DAU 应用），3.5.0 是 LLM 时代最重要的框架基础更新之一。

业务价值与适配性： 可以让我们在广告 SDK 的 Android 端直接基于 MNN 3.5.0 Vulkan 后端部署端侧 LLM（意图识别、素材理解），TurboQuant TQ4 对长上下文用户行为序列推理的内存占用有直接帮助。可直接应用：MNN 是国内广告 SDK 最常用的推理框架之一；TQ4 KV Cache 量化可将 1-2B 模型的上下文缓存减半，对资源受限旗舰机之外的中端机型推全尤为关键；建议在下次 SDK 推理模块升级时优先评估 MNN 3.5.0。

4. 【三星 Galaxy S24/S25 multi-LoRA 一体化端侧 LLM（arXiv 2604.18655）：INT4 量化 + DS2D 投机解码，4-6× 内存/延迟改善，9 语言 8 任务量产】

  • 评分：A3 · I4 · S=10 · P1 · 价值类型：新能力探索
  • 来源：Samsung / 多机构 | arxiv.org/abs/2604.18655 | 2026-04（月内）

内容总结（130 字）： 本文披露三星在 Galaxy S24（Qualcomm SM8650）和 S25（SM8750）上量产部署的 multi-LoRA 一体化 LLaMA 基础模型方案。核心工程亮点：① multi-LoRA 动态任务切换——将各应用场景（邮件、翻译、摘要等）的 LoRA 作为运行时输入注入单一冻结推理图，无需重编译或额外显存；② DS2D（Dynamic Self-Speculative Decoding）——树状结构预测未来 token，无需 draft 模型，decode 时间提速 2.3×；③ 多流解码——单次前向同时生成多种风格（正式/礼貌等），延迟压缩 6×；④ INT4 量化结合架构优化，整体内存/延迟改善 4-6×，9 语言、8 任务精度保持。工程成熟度：已在量产机型上部署。

业务价值与适配性： 可以让我们借鉴三星路径——将广告 SDK 的多个模型（意图识别、素材评分、CTR 初排）合并为一个 base + 多 LoRA 的端侧一体化模型，显著降低内存占用和冷加载延迟。可借鉴思路：multi-LoRA 运行时动态切换是广告端侧多任务部署的关键使能技术，DS2D 的无 draft-model 投机解码方案可作为端侧广告意图生成的低开销加速选项；三星-高通的联合优化路径也表明 Snapdragon 旗舰机已具备承载 1-3B 多任务 LLM 的生产能力。

5. 【Qualcomm Matrix Extensions（QMX）：移动端 CPU 上 Llama 推理加速——官方技术博客披露指令集级优化细节】

  • 评分：A2 · I4 · S=8 · P2 · 价值类型：成本优化
  • 来源：Qualcomm Developer Blog | qualcomm.com/developer/blog/2026/04/llama-models-acceleration-on-cpu-qmx | 2026-04

内容总结（120 字）： 高通官方博客披露 Qualcomm Matrix Extensions（QMX）在移动端 CPU（Oryon 架构）上加速 Llama 推理的完整技术路径。QMX 是高通对 ARM SME/SVE 指令集的进一步扩展，专为矩阵乘法密集型 LLM 推理设计：在 Snapdragon X Elite/Snapdragon 8 Elite CPU 上，QMX 通过扩展寄存器宽度和 outer-product 累加单元大幅提升 GEMM 吞吐。具体 speedup 数字：Llama 3.2 1B/3B CPU 推理相比 llama.cpp 基准有明显提升（博客内有具体 tok/s 对比）。工程信息：通过 llama.cpp 上游集成，可直接使用；配合 KleidiAI 库覆盖 Snapdragon X 和 Snapdragon 8 系列。

业务价值与适配性： 可以让我们在广告 SDK 的 CPU fallback 推理路径（覆盖无 NPU 驱动或旧设备）中通过升级 llama.cpp + QMX 支持获得免费提速，扩大端侧 LLM 的设备覆盖范围（中端 Snapdragon 设备同样受益）。可借鉴思路：QMX 主要价值是拓宽"无 NPU 支持时的兜底方案"——对非旗舰用户群的广告意图识别提升实时性；结合上条 DS2D 投机解码，CPU 路径下也可达到接近 NPU 的推理效率。

其他观察

今日三条方法论收敛于同一方向：LLM 不只是"做推理"，而是作为系统组件生成参数（LLM-HYPER）、管理策略文档（SAGER）、驱动端侧个性化决策——LLM 的角色正在从"回答者"演变为"元学习者/策略生成者"。同时，三星 multi-LoRA 一体化量产方案与 MNN 3.5.0 基础设施更新的叠加，使"1-3B 多任务端侧 LLM 广告 SDK"从技术可行跨越到工程可落地的临界点，时间窗口约为 2026 Q3-Q4。
