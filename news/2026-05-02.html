2. 【KVTC（ICLR 2026）：KV Cache 变换编码——PCA 去相关 + 自适应量化 + 熵编码，20–40× 压缩比，长上下文精度无损，正在维也纳展示中】

  • 评分：A/3 · I/3 · S=9 · P1 · 价值类型：成本优化
  • 来源：多机构 | arxiv.org/abs/2511.01815 | ICLR 2026 正式接受（5/1–5 维也纳展示中）

内容总结（~130 字）： KVTC 是轻量变换编码器，针对 LLM KV Cache 的结构冗余，以三步流水压缩：① PCA 特征去相关，对 Key/Value 向量做主成分旋转消除通道相关性；② 自适应量化，按通道重要性动态分配 bit 数；③ 熵编码，进一步压缩符号熵。无需修改注意力计算或模型权重，仅需简短校准，与现有服务架构完全兼容。在 Llama 3、Mistral NeMo、R1-Qwen 2.5 上验证：20× 压缩维持 AIME25、GSM8K、MMLU、LongBench 精度无显著下降；40× 压缩在特定短上下文场景可用。与 token 驱逐、SVD 等基线相比，在相同压缩比下精度更高；与 MTServe（05-01 报道的分层 KV Cache 溢出方案）正交可叠加。

业务价值与适配性： 可以让我们在广告生成式精排服务中将用户长历史序列 KV Cache 的 GPU 显存占用压缩至原来的 1/20–1/40，在同等硬件下支持 10–20× 并发量提升，直接降低每次用户 session 的服务成本。可借鉴思路：KVTC 与 MTServe 可组合部署——KVTC 先在 GPU 内 20× 压缩热 Cache，MTServe 再将低频 Cache 溢出到 RAM；两层叠加理论上释放 >95% 显存，建议在内部生成式推荐 POC 中纳入联合工程测试。

3. 【ZipperQuant（ICLR 2026）：GPU+CPU 异构 4-bit 量化——离群值高位比特卸载至 CPU AVX2 查表加速，3.01× 推理提速，近 FP16 精度】

  • 评分：A/3 · I/3 · S=9 · P1 · 价值类型：成本优化
  • 来源：多机构 | openreview.net/forum?id=eHb35nP5zC | ICLR 2026 正式接受（5/1–5 维也纳展示中）

内容总结（~130 字）： 4-bit 量化的核心难题是离群值权重（outliers）——少量通道数值远大于均值，强制 INT4 精度损失集中于此。ZipperQuant 以"比特层位分离"范式解决：① GPU 执行低阶比特（所有 inliers + outliers 低位），走标准 INT4 计算路径；② CPU 仅处理 outliers 高阶比特，通过预计算 LUT（查表）替代乘法，配合 Intel AVX2 向量指令消除反量化开销。实测：RTX 4090 上相比 W4A16 基线加速 3.01×，精度接近 FP16。CPU 端只需标准 Intel AVX2 支持，任何配备 Intel CPU + GPU 的服务器无需额外硬件即可部署，适合现有广告推理机群的 drop-in 升级。

业务价值与适配性： 可以让我们在广告云侧 LLM 排序/改写服务中通过 ZipperQuant 将现有 W4A16 推理吞吐提升 3×，在不新增 GPU 机器的前提下承载 3× 的广告请求量，或将同等请求量的服务成本压缩至 1/3。可借鉴思路：ZipperQuant 与 ZeroQAT（05-01 报道，端侧量化感知训练）互补——ZeroQAT 确保 INT4 权重精度，ZipperQuant 确保 INT4 推理速度；两者结合构成"高精度 INT4 训练 + 高速 INT4 服务"的完整云侧 LLM 量化栈。

4. 【LittleLamb（Multiverse Computing，HuggingFace，2026-04-28）：量子启发张量网络压缩——0.3B 三变体，95% 体积缩减、2–3% 精度损失，支持工具调用与移动端部署】

  • 评分：A/2 · I/3 · S=7 · P2 · 价值类型：新能力探索
  • 来源：Multiverse Computing | theaiinsider.tech/2026/04/28/multiverse-computing-littlelamb-models-hugging-face | 2026-04-28

内容总结（~120 字）： Multiverse Computing（量子计算创业公司）发布 LittleLamb 0.3B 系列，采用专有 CompactifAI 技术——量子启发张量网络数学对 LLM 进行后训练压缩：将模型权重分解为低秩张量乘积，相比传统 INT4/INT8 量化机制本质不同（张量分解 vs. 数值精度降低）。压缩效果：高达 95% 体积缩减、2–3% 精度损失。三个变体：LittleLamb-0.3B（通用）、Tool-Calling（支持工具调用和 Agentic 工作流）、Mobile（针对移动端和离线设备优化）。全部 HuggingFace 开源（Apache 2.0），目标移动/边缘/离线场景，低延迟低资源消耗。

业务价值与适配性： 可以让我们在 Android 广告 SDK 中将意图识别/查询改写模块压缩至 0.3B 级（模型约 150MB），实现全端侧推理不依赖云端，覆盖网络质量差的用户群体。暂不适用于生产：CompactifAI 精度损失（"2–3%"）缺乏广告场景第三方独立验证；LittleLamb-Mobile 的 Android/iOS 推理框架适配方案未披露；建议列入"超轻量端侧 SLM 技术雷达"，关注后续第三方 CTR/意图识别 benchmark 和 MNN/ExecuTorch 框架集成进展。

其他观察

今日两大线索收敛于同一方向：基础设施层（KVTC + ZipperQuant）以不同维度压缩 LLM 推理成本——一个在显存维度（20–40×），一个在计算维度（3×）——印证"生成式推荐/排序的边际服务成本将在 2026 下半年大幅下降"的判断；应用生态层（Doubao Agentic 智能机）展示广告大厂深嵌 OS 的战略路径。两者叠加：云侧成本降、端侧权限扩，广告 AI 的下半年格局正在成型。
